# say Overview

## Summary

`say` is the baseline testing and onboarding skill for this workspace.

## User Value

It gives the user a simple example of how a Developer Dashboard skill can be:

- installed
- uninstalled
- a CLI command
- a browser page
- a test suite
- a changelog and skill-local documentation set

## Current Features

- `hello` CLI command prints `hello`
- `/app/say/hello` renders `Hello World`
- Playwright browser test verifies the browser page
- Docker-only verification and 100% coverage are enforced

## Intended Scope

`say` is intentionally minimal. It is not meant to be a standalone product feature. Its purpose is to help users try the skill lifecycle and see a working skill command and page in Developer Dashboard.
